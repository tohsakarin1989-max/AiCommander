# AiCommander 升级路线图 TODO

## 总体边界

AiCommander 当前主线定位为“涉油案件数智研判与防控辅助系统”。系统不做犯罪预测，不替代人工研判，不直接派发跨部门执行任务；核心价值是把已侦破、已抓获、已处置的案件和数智自动化事件沉淀成可检索、可解释、可复盘的经验资产。

## 执行顺序

### 阶段 1：数智自动化与技防边界修正

- [x] 将技防联动统一收敛到“数智自动化”模块，不再把它当作普通辖区台账主线。
- [x] 将模拟告警动作从“派发巡逻”改为“生成事件、生成现场核查建议、标记误报/设备异常、转案件”。
- [x] 页面文案明确真实 A2、雷达、云台、视觉识别仍为待接入能力。
- [x] 事件记录中保留设备异常、AI 研判摘要、核查建议和人工复核结果。

### 阶段 2：大模型调用链口径校准

- [x] 梳理所有 LLM 入口：案件预处理、助手、结论工厂、地图位置研判、圆桌/多模型复核、Agent、向量检索。
- [x] 调整案件预处理 schema，删除或降级“团伙结构、完整销赃链条、巡逻派发、圆桌会议”等不稳定字段。
- [x] 新增事实、推断、建议三层输出，要求 LLM 必须标注依据，不得把推断写成事实。
- [x] 对无模型、模型失败、JSON 解析失败路径保持确定性降级。

### 阶段 3：案件研判工作台与大模型贯通

- [x] 将 `case_intelligence` 的相似条件、现场要素、风险区域画像、复盘卡接入智能助手上下文。
- [x] 结论工厂优先引用结构化研判结果和证据依据，再由 LLM 生成可读结论。
- [x] 报告生成区分“事实依据、模式发现、防控参考、信息缺口”。
- [x] 页面上显示每条建议来自哪些案件、标签或现场条件。

### 阶段 4：数智自动化后端实体化

- [x] 建立自动化告警数据结构：井口参数异常、设备来源、雷达/云台摘要、AI 研判结论、人工核查结果。
- [x] 提供告警列表、详情、事件转换、误报归档、案件转换 API。
- [x] 支持将数智自动化告警作为案件来源和相似条件分析依据。
- [x] 保留外部系统接入适配层，真实接入前可使用模拟数据。

### 阶段 5：研判问答与复盘体验升级

- [x] 助手支持回答“为什么这个区域高风险”“这起案件和哪些案件条件类似”“建议依据是什么”。
- [x] 案件详情中增加复盘入口，展示字段质量、标签、相似条件、经验卡。
- [x] 数智自动化事件转案件后自动触发预处理和研判摘要生成。
- [x] 输出报告支持复制 Markdown，并保留数据来源引用。

### 阶段 6：回归验证与质量门禁

- [x] 后端新增 API 契约测试，覆盖数智自动化、预处理降级、研判报告生成。
- [x] 前端通过 `npm run typecheck` 和 `npm run build`。
- [x] 后端通过 `pytest`。
- [x] 对关键页面保留手工测试步骤：数智自动化、案件研判、助手、结论工厂。

### 阶段 7：模型上下文包

- [x] 案件研判工作台新增大模型上下文包，集中输出事实依据、模式推断、防控参考、信息缺口和证据索引。
- [x] 上下文包生成可复制提示词，明确大模型只能辅助解释，不替代评分、处置和人工研判。
- [x] 前端新增“模型上下文”页签，支持复制提示词并查看事实、推断、建议、缺口和边界。

### 阶段 8：研判辅助 Agent

- [x] 将 Agent 口径从“自动侦查”调整为“研判辅助”，禁止输出自动派发、犯罪预测和未掌握链条。
- [x] Agent 后端接入案件研判上下文包，输出 steps、facts、inferences、recommendations、information_gaps、evidence_refs。
- [x] 前端支持选择案件、使用预置研判目标，并结构化展示事实依据、模式推断、防控参考和信息缺口。

### 阶段 9：数智告警研判包

- [x] 数智自动化告警新增研判包接口，汇总告警事实、AI 依据、信息缺口、下一步和边界。
- [x] 告警转案件后研判包自动挂接案件研判上下文，形成“告警 -> 事件 -> 案件 -> 复盘”的闭环。
- [x] 前端告警队列新增“研判包”入口，可直接跳转到案件研判工作台。

## 手工测试步骤

1. 数智自动化：进入 `/intelli-inspect`，确认模拟告警来自后端；依次点击“生成事件”“核查建议”“标记误报”“转案件”，确认没有“派发巡逻”入口。
2. 案件研判：从案件列表点击“案件研判”图标，确认跳转 `/case-intelligence?caseId=...` 后选中对应案件，并展示标签、相似条件、建议来源和报告。
3. 智能助手：询问“这起案件和哪些案件条件类似”“建议依据是什么”，确认回答使用案件研判工作台上下文，并区分事实、推断、建议。
4. 结论工厂：对案件生成结论，确认 `evidence.raw.case_intelligence` 包含复盘卡、相似案件、防控建议和研判报告。

# 指挥大屏重设计规格

**日期**：2026-05-10  
**状态**：设计已确认，待实现计划  
**确认稿**：B1-C v4，三栏并列、趋势优先、地图可放大缩小、列表慢速自动轮播  
**适用页面**：`frontend/src/pages/Dashboard/Dashboard.tsx` 与 `frontend/src/pages/Dashboard/Dashboard.css`

---

## 背景

当前项目定位是“涉油案件数智研判与防控辅助系统”。指挥大屏要服务领导快速掌握趋势、重点区域、链条推断、AI 产出和奖金考核材料缺口，不能扩展成巡逻执行平台或通用综合管理平台。

现有 Dashboard 已具备可复用基础：

- SVG 地图、热点、缩放、拖拽、复位和全屏能力；
- WebSocket 实时案件统计与案件列表；
- `caseApi.getHotspots` 提供热点数据；
- `caseApi.getChainMapData` 对接 `GET /api/chain-links/map-data`；
- 链条接口已返回边界说明：地图连线仅表示系统推断或人工确认的上下游关联，未确认连线不得直接作为定案依据。

本次设计是在这些基础上重排信息架构，让大屏从“地图主导的实时态势”升级为“领导研判视图”：左侧看趋势，中间看空间与链条，右侧看 AI 产出与待复核工作。

---

## 设计目标

1. **领导可读**：第一屏直接看到趋势、重点、产出和待复核事项。
2. **数据有出处**：地图和指标必须标明数据口径，不展示无法追溯的“装饰性结论”。
3. **无人值守可展示**：列表内容慢速自动轮播，不依赖人工滚动。
4. **临场可放大**：地图支持放大、缩小、复位、拖拽平移，便于现场查看点位和关系线。
5. **项目风格一致**：使用现有 `design-system.css` 的深色工作台变量、琥珀强调色、线框卡片和高信息密度布局。

---

## 页面结构

页面沿用现有 App Shell 顶栏，品牌名保持：

- 主品牌：`涉油案件指挥系统`
- 副标识：`AiCommander · 实时`

Dashboard 页面内部采用三段结构：

1. **顶部系统栏**：复用现有导航、在线状态、当前用户角色信息。
2. **摘要 KPI 行**：页面标题 + 5 个关键指标。
3. **三栏研判面板**：左趋势、中地图、右产出。

### 摘要 KPI

KPI 行包含：

- 本月案件：来自案件统计或 WebSocket `statistics.this_month_cases`；
- 高风险区域：来自 `patrolApi.getAreaRisks` 或热点风险聚合；
- 链条推断：来自 `caseApi.getChainMapData` 的总数及待确认数量；
- AI 报告产出：来自圆桌报告、部署建议、经验卡等已有业务数据；缺少统计源时显示解释性空态；
- 材料齐全率：来自案件质量/奖金考核材料完整性计算，缺数据时显示“待补录”而不是编造百分比。

### 三栏布局

采用统一父级 CSS Grid，保证三栏行高对齐：

- 左列：`案件趋势`、`风险变化`、`考核材料趋势`
- 中列：`空间分布与链条关系` 跨前两行，`本周研判重点` 占第三行
- 右列：`AI 研判产出`、`待复核事项`、`系统产出质量`

中列底部“本周研判重点”必须与左右第三行同线，解决前一版“中部与两边不齐”的问题。

## 地图设计

地图模块标题为：`空间分布与链条关系`。右侧小字显示数据来源：`案件坐标 / 链条接口`。

### 地图内容

地图显示四类信息：

- **事实点位**：来自案件 `latitude` / `longitude`，按链条位置渲染不同形状；
- **热点区域**：来自近 30 天案件密度聚合或 `caseApi.getHotspots`；
- **推断关系**：来自 `GET /api/chain-links/map-data` 的链条连线；
- **缺口提示**：来自缺坐标案件、材料完整性和奖金考核材料检查。

图例固定展示：

- 事实点位
- 推断关系
- 待核坐标

地图底部固定展示产出口径：

- `坐标=案件经纬度`
- `热区=30天密度`
- `关系=链条接口`
- `缺口=坐标/材料`

### 地图交互

正式实现复用当前 Dashboard 的 SVG `viewBox` 缩放和平移能力：

- `+`：放大；
- `-`：缩小；
- `复位`：回到默认视野；
- 缩放比例：显示当前倍率；
- 鼠标拖拽：平移地图；
- 鼠标滚轮：保留当前已有的绕鼠标缩放能力。

地图关系线必须区分状态：

- `confirmed`：实线或高亮线；
- `inferred`：虚线或低饱和线；
- `rejected`：不展示。

未确认链条必须保留边界说明，不得在 UI 文案中表达为“已查实”。

## 自动轮播列表

以下模块使用慢速自动轮播：

- 风险变化；
- 考核材料趋势；
- AI 研判产出；
- 待复核事项；
- 系统产出质量。

轮播规则：

- 每个模块内部使用纵向滚动轨道；
- 一轮时长 42-50 秒，按模块内容量略有差异；
- 鼠标悬停时暂停，移出后继续；
- 顶部和底部使用渐隐遮罩，避免内容突兀裁切；
- 支持 `prefers-reduced-motion: reduce`，用户系统要求减少动态时停止自动轮播，改为静态可滚动列表。

大屏无人值守时，列表能自动展示所有条目；现场操作人员仍可悬停暂停阅读。

## 数据口径

### 案件趋势

来自案件 `occurred_time`，按最近 7 周聚合。柱状图颜色含义：

- 普通周：信息色；
- 风险升温周：琥珀色；
- 需要关注周：红色。

升温判断第一期使用同环比增幅，后续可接入更复杂的趋势算法。

### 风险变化

来自热点、区域风险和近 30 天案件密度变化。每条卡片必须说明判断原因，例如“案件密度上升”“夜间占比增加”“连续两周无新增”。

### 考核材料趋势

来自案件质量与奖金考核相关材料检查。重点展示：

- 检斤含水材料；
- 人员处理结果；
- 车辆移交单据；
- 缺坐标对空间研判的影响；
- 材料齐全后进入奖金测算的状态。

缺口提醒只提示“影响奖金复核/测算”，不替代人工审核。

### AI 研判产出

来自系统内 AI 产物统计：

- 圆桌会议报告；
- 部署建议草案；
- 经验卡；
- 事实/推断/建议分层结果。

若某类数据暂未接入正式统计，显示空态或“暂无本周产出”，不得使用固定假数字。

### 待复核事项

聚合以下待办：

- 链条推断待确认；
- 奖金测算待复核；
- 结论分层待确认；
- 坐标补录待处理；
- 材料缺口待补齐。

待复核事项只做提醒和入口，不在大屏上执行确认、驳回、计算或提交。

### 系统产出质量

展示系统辅助质量指标：

- 事实引用率；
- 推断采纳率；
- 材料缺口命中率；
- 报告退回率。

第一期若缺少完整后端统计，可由前端从已加载数据推导可解释指标；无法推导时显示空态。

---

## 本周研判重点

中部底部固定展示三项：

- 重点区域；
- 重点链条；
- 待补数据。

来源优先级：

1. 风险变化中的最高优先级区域；
2. 链条接口中置信度最高且未驳回的链条；
3. 缺坐标、缺材料或待复核数量最高的数据缺口。

该模块用于领导快速抓重点，不承载长文本报告。

---

## 交互边界

本次大屏不做以下事情：

- 不自动派发巡逻任务；
- 不在大屏上确认或驳回链条推断；
- 不在大屏上提交奖金考核；
- 不把未确认推断表达为事实；
- 不引入新的第三方地图服务或 API Key；
- 不上传或展示项目外汇报材料、申报材料等隐私文件。

大屏是“研判展示与复核提示”界面，不是处置执行界面。

## 实现边界

正式实现应优先在现有文件内完成：

- `frontend/src/pages/Dashboard/Dashboard.tsx`
- `frontend/src/pages/Dashboard/Dashboard.css`

可新增小型本地组件或工具函数，但必须保持职责清晰：

- 自动轮播列表组件：只负责展示和暂停；
- 指标聚合工具：只负责把案件、链条、质量数据转成 Dashboard 可用的视图模型；
- 地图渲染仍留在 Dashboard 地图区域，避免过早拆成复杂地图框架。

后端接口第一期只读取现有能力，不新增数据库字段。若实现时发现 AI 产出质量、材料齐全率没有稳定聚合接口，先在前端显示解释性空态或从已加载案件质量数据中推导，不阻塞整体大屏重排。

## 验收标准

1. 页面在 `localhost:3000/dashboard` 中显示新三栏大屏布局。
2. 顶栏、标题、卡片、颜色与现有 `design-system.css` 风格一致。
3. “本周研判重点”与左右第三行面板对齐。
4. 列表模块自动慢速轮播，悬停暂停。
5. 地图可以放大、缩小、复位和拖拽平移。
6. 地图展示产出口径，不出现无来源的装饰性结论。
7. 未确认链条以推断状态展示，不表达为查实事实。
8. 数据缺失时有明确空态，不显示虚假固定值。
9. `npm run build` 通过。
10. 浏览器截图检查无重叠、无窄条、无主要文字溢出。

- `cd frontend && npm run build`

## 风险与处理

- **数据源不足**：AI 产出质量和材料齐全率可能没有现成聚合接口。处理方式是先显示可解释空态或前端推导指标，不写死假数据。
- **大屏动态过多**：自动轮播过快会影响阅读。处理方式是每轮 42-50 秒，悬停暂停，并尊重系统减少动态设置。
- **地图误导风险**：链条连线容易被理解为事实。处理方式是保留推断/确认状态和边界说明，未确认连线低饱和展示。
- **Dashboard 文件过大**：现有 Dashboard 已较长。实现时可拆出小型纯展示组件，但不做与本任务无关的大重构。

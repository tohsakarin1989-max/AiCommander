# AiCommander 全面升级设计文档

**日期**：2026-04-10  
**状态**：已审批  
**范围**：UI 深色主题化、地图功能升级、时空研判新模块、Prompt 质量优化

---

## 背景与目标

当前系统存在三个核心问题：

1. **地图功能弱**：现有 ECharts 散点图仅为坐标系，无地理底图，无法直观呈现案件空间分布
2. **时空研判薄弱**：后端已有轨迹 API，但前端缺乏专用页面，功能未被充分利用
3. **UI 风格陈旧**：浅色 Ant Design 默认主题，不符合指挥中心专业气质

升级目标：全面实现预期功能，完成页面美化，使系统达到专业公安/安全领域系统标准。

## 二、侧边栏导航

**方案**：手风琴折叠（Accordion），点击一级分组展开/收起子菜单，同一时间只展开一组。

**导航结构**：

```
⬡ AI案件分析系统
├── 首页（Dashboard）
├── [案件分析]  可折叠分组
│   ├── 案件管理
│   ├── 案件地图
│   ├── 区域研判
│   ├── 时空研判  ← 新增
│   └── 串案图谱
├── [会议报告]  可折叠分组
│   ├── 圆桌会议
│   └── 分析报告
└── [智能辅助]  可折叠分组
    ├── AI 助手
    └── 智能体中心
```

**实现要点**：

- 使用 Ant Design `Menu` 组件的 `openKeys` 受控模式实现手风琴（`onOpenChange` 中只保留当前打开的一个 key）
- 侧边栏宽度展开 200px，不做折叠（屏幕足够宽）
- 活动项：左侧 2px `#7dd3fc` 边框 + 淡蓝背景 `rgba(125,211,252,0.12)`
- 顶部 Logo 区域：`⬡ AI案件分析系统`，字色 `#7dd3fc`

## 三、地图功能升级

### 技术选型

| 库 | 版本 | 用途 |
|----|------|------|
| `react-leaflet` | ^4.x | React 封装的 Leaflet |
| `leaflet` | ^1.9 | 核心地图库 |
| `leaflet.markercluster` | ^1.5 | 案件聚合 |
| `leaflet.heat` | ^0.2 | 热力图 |

底图：CartoDB Dark Matter（`https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png`），无需 API Key，需要联网。

### 页面布局（`CasesMap.tsx`）

```
[侧边栏] | [图层控制面板 200px] | [地图主体 flex:1] | [AI分析面板 190px]
```

**图层控制面板（左）**：
- 图层开关：案件热点、管线分布、巡逻路线建议、车辆轨迹
- 底图切换：街道 / 卫星 / 暗色
- 热点区域列表（按风险等级排序）

**地图主体**：
- 案件聚合圆圈：数量标注，颜色 = 最高风险等级（红/黄/绿）
- 点击聚合圆 → 展开；点击单个标记 → Popup 显示案件编号、类型、时间、手法
- 管线图层：黄色虚线 Polyline，第一期使用案件关联的经纬度点连线模拟管线走向（将同一区域内多个案件按距离排序后依次连线），后续可替换为真实管线 GIS 数据
- 串案连线：同组串案标记间用虚线连接，颜色 `#a78bfa`
- 地图搜索框（左上角）：按案件编号/地点名称定位

**AI 分析面板（右）**：
- 区域风险分布（进度条）
- 串案关联摘要
- AI 巡逻建议（前 2 条，优先级排序）
- 底部"查看完整报告"按钮

**底部状态栏**：当前坐标、视野内案件数量、图例

### 地图拾取器（补充，集成到案件表单）

在新建/编辑案件弹窗中，经纬度输入框下方嵌入迷你地图（高度 200px）：

- 初始位置：若已有经纬度则居中显示，否则默认显示大庆市区
- 点击地图任意位置 → 放置标记，同时自动填充纬度/经度表单字段
- 拖动标记 → 实时更新字段值
- 表单字段与地图双向绑定：手动修改数字输入框也更新地图标记位置

## 四、时空研判模块（新增页面）

**路由**：`/cases/spacetime`  
**文件**：`frontend/src/pages/Cases/SpaceTimeAnalysis.tsx`

### 功能区块

**1. 时间轴筛选器（顶部横条）**

- 日期范围选择器（DatePicker.RangePicker）
- 选中后地图和统计图表同步过滤案件

**2. 地图区（左侧 60% 宽）**

- 复用 Leaflet 地图组件
- 新增"轨迹回放"模式：
  - 按发案时间顺序，依次高亮标记（带动画）
  - 播放/暂停/速度控制条（0.5x / 1x / 2x）
  - 时间游标跟随当前播放时间点

**3. 统计面板（右侧 40% 宽）**

- **24 小时发案分布**：ECharts 环形/雷达图，标注高峰时段
- **周发案规律**：ECharts 柱状图（周一~周日）
- **月度趋势**：折线图，近 6 个月走势
- **预测提示**：基于规律，文字输出"下次高风险时段预测"（调用后端 `/api/cases/trajectory/{ids}/predict`）

### 后端 API 对应

| 功能 | API |
|------|-----|
| 获取轨迹数据 | `GET /api/cases/trajectory/{ids}` |
| 时序分析 | `GET /api/cases/trajectory/{ids}/analysis` |
| 下一发案预测 | `GET /api/cases/trajectory/{ids}/predict` |
| 轨迹回放数据 | `GET /api/cases/trajectory/{ids}/replay` |

## 五、Prompt 优化

### analyst.py 修改点

**空间专家（specialty=spatial）**：

在区域聚集特征分析中，明确将以下设施类型列为重点空间对象：
- 输油管线（干线/集输支线）沿线
- 油库/储油罐区周边
- 加油站密集区段
- 罐车常用停靠路段

**手法专家（specialty=modus）**：

在作案模板提炼中，明确列举涉油案件常见手法：
- 打孔盗油（在管线上打孔，使用软管抽取）
- 切割管线（截断管段，整体运走）
- 偷接管线（非法接驳分支管道）
- 罐车过驳（用罐车直接装载盗取的油品）
- 混入合法装运（伪造运输单据）

**防控专家（specialty=prevention）**：

联防联控维度新增：
- 与管道运营公司/油田保卫部门的联防机制建议
- 是否需要启动管线巡检频次调整

### moderator.py 修改点

`generate_final_report` JSON schema 新增两个字段：

```json
{
  "risk_trend": {
    "direction": "increasing/stable/decreasing",
    "description": "近期发案趋势描述"
  },
  "infrastructure_risks": [
    {
      "facility_type": "设施类型（管线/油库/加油站）",
      "risk_level": "high/medium/low",
      "description": "具体风险描述"
    }
  ]
}
```

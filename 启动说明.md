# AI 案件分析系统 - 启动说明

## 快速启动

### 方式一：Docker 启动（推荐）

```bash
# 启动（需要先打开 Docker Desktop）
./start.sh

# 停止
./stop.sh
```

### 方式二：本地开发模式

需要打开 **两个终端窗口**：

#### 终端 1 - 启动后端

```bash
cd backend
source venv/bin/activate
export SECRET_KEY=your-secret-key
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 终端 2 - 启动前端

```bash
cd frontend
npm run dev
```

## 首次运行

如果是首次运行项目，需要先完成以下准备工作：

### 后端准备

```bash
cd backend
python -m venv venv              # 创建虚拟环境
source venv/bin/activate         # 激活虚拟环境
pip install -r requirements.txt  # 安装依赖
python init_db.py                # 初始化数据库
```

### 前端准备

```bash
cd frontend
npm install  # 安装依赖
```

## 环境变量配置

### 必需配置

在 `backend/.env` 文件中配置：

```env
SECRET_KEY=your-secret-key-here
```

### 可选配置（AI 功能）

```env
# OpenAI 模型
OPENAI_API_KEY=sk-xxx

# Anthropic 模型
ANTHROPIC_API_KEY=sk-ant-xxx

# 数据库（默认使用 SQLite）
DATABASE_URL=sqlite:///./aicommander.db

# Redis（默认本地）
REDIS_URL=redis://localhost:6379/0
```

## 常见问题

### 1. Docker 未运行

```
错误: Docker未运行，请先启动Docker
```

**解决：** 打开 Docker Desktop 应用，等待启动完成后再试。或使用本地开发模式。

### 2. SECRET_KEY 错误

```
pydantic_core._pydantic_core.ValidationError: 1 validation error for Settings
SECRET_KEY
  Field required
```

**解决：** 设置环境变量 `export SECRET_KEY=your-secret-key` 或创建 `.env` 文件。

### 3. 端口被占用

```
ERROR: [Errno 48] Address already in use
```

**解决：**
```bash
# 查找占用端口的进程
lsof -i :8000
# 终止进程
kill -9 <PID>
```

### 4. 前端依赖问题

```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
```

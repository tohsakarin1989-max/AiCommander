# 使用说明

## 系统功能

### 1. AI模型配置
- 在"AI模型配置"页面添加和管理AI模型
- 支持OpenAI和Anthropic提供商
- 可以配置主持人模型和分析员模型
- 支持测试模型连接

### 2. 案件管理
- 在"案件管理"页面添加和管理案件信息
- 包括案件编号、发生时间、地点、类型、描述等

### 3. 圆桌会议
- 在"圆桌会议"页面创建会议
- 选择要分析的案件
- 选择主持人模型和分析员模型
- 系统会自动执行以下流程：
  1. 主持人格式化案件信息
  2. 各分析员独立分析
  3. 分析员匿名互评
  4. 基于反馈优化分析
  5. 主持人汇总生成最终报告

### 4. 分析报告
- 在"分析报告"页面查看已完成会议的报告
- 报告包含共识点、分歧点、建议等

## 使用流程

1. **配置AI模型**
   - 进入"AI模型配置"页面
   - 添加至少一个主持人模型和多个分析员模型
   - 测试模型连接确保可用

2. **添加案件**
   - 进入"案件管理"页面
   - 添加需要分析的案件信息

3. **创建会议**
   - 进入"圆桌会议"页面
   - 点击"创建会议"
   - 选择案件、主持人模型和分析员模型
   - 系统会自动开始分析

4. **查看结果**
   - 在会议列表中查看会议状态
   - 点击"查看"查看会议对话和最终报告

## 注意事项

1. **API密钥安全**
   - API密钥会加密存储
   - 请妥善保管API密钥

2. **会议执行时间**
   - 会议分析可能需要较长时间
   - 请耐心等待，可以在会议列表中查看进度

3. **模型选择**
   - 建议使用性能较好的模型作为主持人
   - 分析员可以使用不同模型以获得多样化观点

4. **数据库**
   - 首次运行需要初始化数据库
   - 使用Docker时数据库会自动创建

## 故障排除

### 后端无法启动
- 检查数据库和Redis是否运行
- 检查环境变量配置
- 查看日志：`docker-compose logs backend`

### 前端无法连接后端
- 检查后端是否运行在8000端口
- 检查CORS配置
- 查看浏览器控制台错误

### 会议执行失败
- 检查AI模型配置是否正确
- 检查API密钥是否有效
- 查看后端日志
